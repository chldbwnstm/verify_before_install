# 사용자용 리포트 포맷 (한국어)

비개발자(코드를 읽지 못하는 사용자)에게 그대로 보여줄 형식이다.
판정 결과 JSON을 받아 아래 형식으로 변환한다. 기술 용어는 한 줄 비유로 풀어 쓴다.

## 한 줄 형식

각 패키지를 한 줄씩, 판정 기호로 시작한다.

```
🟢 안전 — <패키지> : <한 줄 이유>
🟡 주의 — <패키지> : <한 줄 이유> → <권장 조치>
🔴 차단 — <패키지> : <한 줄 이유> → 설치를 멈췄습니다. 진행하려면 승인이 필요합니다.
```

- 🟢 **안전** : 공식 패키지 목록(앱스토어 같은 공식 배포처)에 실제로 있고 평판도 정상.
  정확한 버전 그대로(정품 봉인=해시 확인) 설치 — '잠금파일'=버전 고정, '해시'=정품 봉인 확인.
- 🟡 **주의** : 신생/다운로드 적음/공개 코드 보관소 없음/검증 불가/보안 경고 등 애매함. 사용자 확인 필요.
- 🔴 **차단** : 존재하지 않음(환각 의심) / 사칭 의심 / 받는 곳을 헷갈린 경우 등. 설치 중단.

## 기호의 뜻 (사용자에게 풀어줄 비유)

- 🟢 = "신원조회 통과한 사람" — 들여보내도 됨
- 🟡 = "신분증이 좀 이상한 사람" — 한 번 더 확인하고 들여보낼지 결정
- 🔴 = "위조 신분증이거나 명단에 없는 사람" — 일단 막음

## 전체 리포트 예시

```
📦 패키지 안전 점검 결과

🟢 안전 — requests : 사전 승인(화이트리스트)된 패키지이고 알려진 보안 취약점이 없습니다.
🟡 주의 — cool-logger : 평판 신호가 약합니다: 신생(12일), 공개 코드 보관소(GitHub 같은 출처) 없음. → 공식 문서나 출처를 확인하고 정말 필요한 패키지인지 점검할게요.
🔴 차단 — image-utils-pro : Python(pip) 공식 패키지 목록(앱을 받는 공식 앱스토어 같은 곳)에 이런 이름이 없습니다(AI가 이름을 지어냈을 가능성). → 설치를 멈췄습니다. 진행하려면 승인이 필요합니다.

※ 이 중 하나라도 🔴/🟡가 있으면 제가 임의로 설치하지 않습니다.
```

> 위 문구는 모두 `verify_packages.py --report` 가 실제로 출력하는 그대로다. 직접 다시 쓰지 말고 스크립트 출력을 그대로 사용자에게 보여주면 된다.

## 필수 규칙

- 마지막 줄에 **항상** 다음 문장을 넣는다:
  `※ 이 중 하나라도 🔴/🟡가 있으면 제가 임의로 설치하지 않습니다.`
- 🔴/🟡가 하나라도 있으면 설치 명령을 실행하지 말고, 사용자에게 명시적 승인을 먼저 요청한다.
- 🔴는 사용자가 "위험을 이해했고 그래도 진행한다"고 명시적으로 답하기 전에는 절대 진행하지 않는다.
- 판정 이유는 한 문장으로, 코드 모르는 사람도 이해할 수 있게 쓴다.

> 참고: `scripts/verify_packages.py --report ...`(경로는 **스킬 폴더 기준**) 를 쓰면 위 형식의 텍스트를
> 그대로 출력해 준다. 프로젝트 루트에서 실행할 땐 스킬 폴더 절대경로를 앞에 붙인다.
